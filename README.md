 ## las-utils
 
 Сейчас в пакете одна утилита: **`las-thin`** — потоковое прореживание большого `.las` через **воксельную сетку** (оставляет **≤1 точки на воксель**) с целью сильно уменьшить размер файла.
 
 ### Установка
 
 ```bash
 python -m pip install -e ".[dev]"
 ```
 
 ### Быстрый старт
 
 - Проредить фиксированным вокселем 1 метр:
 
 ```bash
 las-thin in.las out.las --voxel 1.0 --mode xyz
 ```
 
 - Проредить только по XY (сохраняет рельеф/покрытие, но срезает плотность по горизонтали):
 
 ```bash
 las-thin in.las out.las --voxel 0.5 --mode xy
 ```
 
 - Автоподбор `--voxel` под целевой размер (примерно):
 
 ```bash
 las-thin in.las out.las --target-mb 500 --mode xy
 ```
 
 ### Важные параметры (практика)
 
 - `--chunk-points`: если память упирается — уменьшай (например 200000..500000).
 - `--bloom-mb`: больше — меньше ложных срабатываний Bloom (и чуть больше итоговых точек), но больше RAM.
 - `--mode xy`: часто даёт более “визуально устойчивую” редукцию плотности на больших площадях.
 
 ### Ограничения
 
 - Выход сейчас **только LAS** (без LAZ-сжатия).
 - Автоподбор размера — **приблизительный** (цель: быстро попасть в порядок величины). Если нужен более точный размер — обычно проще 1 раз подобрать `--voxel` и затем фиксировать его.
